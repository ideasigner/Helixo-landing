功能

每天都会被用到的功能

保存即刷新

文件系统监听，VS Code 保存文件，Helixo 预览自动更新。不需要切换窗口，不需要按刷新。

四种导出格式

PDF、PNG、SVG、Figma。都在导出菜单里，不需要开浏览器、不需要截图工具。

PDF
矢量文字
PNG
2x 高清
SVG
可编辑
Figma
DOM 保留

分栏视图

左右双面板，可同时预览两个 HTML 文件。可拖拽调整分栏宽度，互不干扰。

文件树 + 多标签页

打开文件夹自动过滤 HTML 文件。多个文件用标签页管理，支持拖拽排序和关闭。

专注模式 + 快速编辑

F11 全屏专注模式。快速编辑模式可直接在预览区修改 HTML，保存后自动刷新。

缩放控制 + 状态栏

25%~400% 缩放，Cmd+滚轮快捷缩放。状态栏显示文件大小、修改时间、当前缩放比例。

技术细节

说说技术，省得你问我

纯原生技术栈：Vanilla JavaScript + CSS，无 React/Vue，无构建工具，启动快、内存占用低

Electron 33 + Context Isolation：安全沙箱，渲染进程无法直接访问文件系统

文件系统监听：chokidar 递归监听，文件变化实时同步到文件树

导出能力：PDF（原生打印）、PNG/SVG（Canvas 渲染）、Figma（html2canvas + Figma API）

多语言：EN / 中文 / 日本語 / 한국어

常见问题

Helixo 会替代 VS Code 吗？
不会。Helixo 是配套工具，不是竞品。你在 VS Code 写代码，在 Helixo 看效果和导出。
支持 Windows 吗？
目前只有 macOS 版，Windows 在计划中。
免费吗？
免费，开源在 GitHub。
为什么叫 Helixo？
Helix 是 DNA 双螺旋。代码和设计稿，像两条链缠绕在一起。-ixo 来自 Felix 的词根。
导出的 Figma 文件是什么样？
HTML 被转成图片后导入 Figma，保留视觉样式。文字目前为图片形式，支持进一步手动调整。
