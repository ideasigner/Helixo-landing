01

现状与痛点

四个核心问题导致「改 A break B」的回归成本持续高于开发成本。

🗃
文件规模失衡

DOM 事件、业务逻辑、状态管理、UI 更新全部杂糅在 renderer.js 中。改一处功能需要在 5000+ 行中跨多个逻辑区域跳跃。

renderer.js ~5300 行
🔍
隐式全局依赖

大量函数通过闭包变量和 DOM querySelector 隐式获取状态。AI 需要全文搜索才能确认调用方和数据流。

调用链不可追踪
💬
事件链路分散

iframe 与 renderer 通信涉及 postMessage、巨型 switch、多处直接回传。新增 inspector 操作需改 3~4 个不相关区域。

改动面不可预测
🛠
无自动化测试

所有验证依赖手工启动 App 点击。最近 3 次迭代均出现回归，修复成本高于开发成本。

0 单元测试

重构目标

按优先级分层，P0 必须完成，P1 强烈建议，P2 锦上添花。

优先级 目标 验收标准

模块拆分方案

从单一巨石文件演进为按业务领域分层的模块树。

目标目录结构

src/renderer/
  index.html              # 不变
  index.css               # 不变
  i18n.js                 # 不变
  helixo.js               # 不变（全局配置/常量）
  utils.js                # 不变（通用工具函数）
  preview.js              # 不变（iframe 内注入脚本）

  main.js                 # 入口：初始化顺序、模块组装、全局快捷键

  tabs/
    controller.js         # Tab 增删改查、状态管理
    renderer.js           # Tab DOM 渲染（tab bar、iframe 创建/复用）
    navigation.js         # 前进后退、链接点击、hash 导航

  preview/
    manager.js            # iframe src 加载、生命周期、刷新控制
    ruler.js              # 标尺宽度计算、applyRulerToGroup、ResizeObserver
    zoom.js               # 缩放值管理、setFrameDocumentZoom、reset
    inspector.js          # 检查器开关、tooltip、context menu、坐标转换

  analyze/
    index.js              # Analyze 面板入口，注册各模块
    colorPalette.js       # 色彩提取、刷新、渲染
    typography.js         # 字体清单提取、刷新、渲染
    breakpoints.js        # 断点提取、渲染、点击事件

  fileTree/
    renderer.js           # 树形 DOM 渲染
    interactions.js       # 点击、拖拽、右键菜单
    search.js             # 过滤、高亮

  workspace.js            # 已存在：工作区状态序列化/反序列化
  status.js               # 已存在：状态栏消息
  outline.js              # 已存在：大纲渲染
  events.js               # 新增：轻量事件总线

关键模块职责边界

preview/manager.js
管理 iframe 的创建、销毁、src 加载。
禁止：直接操作 ruler width、zoom level、inspector state
export function createPreviewFrame(container, src)
export function loadPreviewFrame(frame, src)
preview/ruler.js
唯一有权修改 iframe 的 width/height/transform 的模块。
禁止：直接调用 zoom 或 inspector 的函数
export function computeRulerStyles(width, cw, ch)
export function applyRulerToGroup(group, width)
preview/inspector.js
管理检查器开关状态、tooltip、context menu、坐标映射。
禁止：直接修改 iframe 样式或 ruler 状态
export function enableInspector(tab)
export function disableInspector(tab)
export function mapInspectorCoords(x, y, iframe)
events.js（事件总线）
轻量级 pub/sub，基于 EventTarget。
禁止：在事件处理器中同步触发新事件（防循环）
export function on(event, handler)
export function emit(event, payload)

事件总线约定

事件名 发布者 订阅者
tab:switched tabs/controller.js preview/inspector.js, preview/zoom.js
ruler:changed preview/ruler.js preview/inspector.js（重算坐标映射）
inspector:contextmenu preview/inspector.js main.js（显示菜单）
inspector:copyImage main.js preview/inspector.js
zoom:changed preview/zoom.js preview/manager.js

05

代码规范约定

四条硬性规则，重构过程中新写的模块必须遵守。

规则 1

模块间调用规则

允许向下调用，禁止横向直接操纵。跨领域通信必须通过事件总线。

✗ analyze/breakpoints.js → preview/ruler.js
✓ analyze/breakpoints.js → emit('ruler:setWidth', 1440)
✓ preview/ruler.js → on('ruler:setWidth', ...)
规则 2

函数签名必须带 JSDoc

每个导出的函数都要有 @param 和 @returns，复杂对象用 @typedef。

✓ @param {Object} group
✓ @param {string|number} width
✓ @returns {RulerStyles}
规则 3

禁止新增隐式全局

新模块不准访问 window.* 上的业务状态。所有依赖通过参数传入或事件总线获取。

✗ function foo() { return window.rulerWidth; }
✓ function foo(width) { return width; }
规则 4

DOM 查询集中化

每个模块在初始化时缓存自己的 DOM 引用，禁止在业务函数中临时 getElementById。

✗ function show() {
  document.getElementById('menu').style.display = '';
}
✓ const menu = document.getElementById('menu');
function show() { menu.style.display = ''; }
